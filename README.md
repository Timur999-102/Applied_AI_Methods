# Ознакомление с темой

**Выбрана тема** - Анализ обфусцированного powershell кода на вредоносность при помощи нейронной сети

## 1. О чем эта тема? (Суть)

PowerShell — это мощный инструмент администрирования в Windows. Злоумышленники активно используют его для атак типа "Living off the Land" (использование легитимных инструментов для вредоносных целей), так как PowerShell уже установлен в системе и имеет высокий уровень доверия.

Обфускация — это процесс намеренного запутывания кода, чтобы скрыть его истинное назначение от антивирусов и аналитиков. Примеры: кодирование в Base64, конкатенация строк, использование псевдонимов команд, вставка мусорных символов.

Нейронная сеть (НС) в данном контексте выступает как классификатор. Она не ищет известные сигнатуры вирусов (как традиционный антивирус), а анализирует структуру, паттерны и семантику скрипта, чтобы определить, является ли он вредоносным, даже если код сильно изменен.

## 2. Для чего и что делает проект?

Цель: Создать систему, которая автоматически детектирует вредоносные PowerShell-скрипты, прошедшие через обфускацию, с высокой точностью и минимальным количеством ложных срабатываний.

Функционал:
1. Прием данных: Получение сырого PowerShell-кода (строки или файла).
2. Предобработка: Частичная деобфускация (например, декодирование Base64), нормализация токенов.
3. Анализ: Извлечение признаков (векторное представление кода).
4. Классификация: Выдача вероятности того, что скрипт вредоносный (Malicious) или легитимный (Benign).
5. Объяснимость (опционально): Подсветка подозрительных участков кода.

## 3. Оценка сложности проекта

Общий уровень сложности: Высокий

Это не простой учебный проект, так как он требует знаний в двух сложных областях.

Сбор данных, сложность: 🔴 Высокая
Нужен сбалансированный датасет (тысячи чистых и грязных скриптов). Публичные датасеты часто устарели.
Предобработка, сложность: 🔴 Высокая
Обфускация бывает очень хитрой. Нужно уметь парсить PowerShell AST (Abstract Syntax Tree), а не просто текст.
Модель ML, сложность: 🟡 Средняя
Архитектуры (LSTM, Transformer) известны, но нужна тонкая настройка под код, а не под естественный язык.
Ложные срабатывания: 🔴 Критично
Если система заблокирует скрипт админа — это инцидент. Требуется высокая точность (Precision).
Инфраструктура, сложность: 🟡 Средняя
Нужен API, возможно интеграция с SIEM/EDR системами.

Главные риски:
Adversarial Attacks: Злоумышленники могут специально подбирать код, чтобы обмануть вашу нейросеть.
Дрейф данных: Техники обфускации меняются быстрее, чем вы переобучаете модель.

## 4. Архитектура проекта

Для такого проекта лучше всего подходит микросервисная архитектура с конвейером обработки данных (Pipeline).

Технологический стек:
* Язык: Python (основной для ML).
* ML Framework: PyTorch или TensorFlow (Keras).
* Обработка кода: Библиотека System.Management.Automation (через Python.NET) или парсеры типа PSParser.
* API: FastAPI (быстро, асинхронно).
* Контейнеризация: Docker.

## 5. План реализации

1. Сбор датасета (Важно: Пометьте, какие техники обфускации использованы.)
2. Базовый анализ (Напишите скрипт на Python, который считает статические признаки (энтропия, длина, наличие ключевых слов).Обучите логистическую регрессию. Оцените точность.)
3. Разработка парсера (Реализуйте токенизацию или извлечение AST. Это критически важно для борьбы с обфускацией.)
4. Обучение Нейросети (Реализуйте архитектуру (например, LSTM).Проведите эксперименты, подберите гиперпараметры.)
5. Деплой и API ) (Оберните модель в Docker-контейнер с FastAPI.)
6. Тестирование на обфускации  (Возьмите чистый вредоносный скрипт и прогоните его через генераторы обфускации (например, Invoke-Obfuscation). Проверьте, детектирует ли ваша модель запутанную версию.)
